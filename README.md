# KabuSys

日本株向けの自動売買（アルゴ）システムのベースとなるPythonパッケージです。コアはモジュール化されており、データ取得、売買戦略、発注実行、監視/ログの各機能を分離して実装できます。

バージョン: 0.1.0

---

## プロジェクト概要

KabuSysは、日本株の自動売買システムを構築するための軽量な骨組み（スケルトン）です。各責務を分けたモジュール構成により、以下を容易に行えます。

- データ取得ロジックの実装（ティッカー、板、約定履歴など）
- 売買戦略（シグナル生成）の実装
- 発注（API連携 / シミュレーション）処理の実装
- 監視・ログ・メトリクスの実装

現状はパッケージ構造のみ提供しており、各モジュールの実装を拡張して使用します。

## 機能一覧（現状）

- パッケージ骨組み（data / strategy / execution / monitoring）の提供

## セットアップ手順

1. リポジトリをクローン（例）
   ```
   git clone <リポジトリURL>
   cd <リポジトリ>
   ```

2. 推奨：仮想環境を作成して有効化
   - macOS / Linux
     ```
     python3 -m venv .venv
     source .venv/bin/activate
     ```
   - Windows (PowerShell)
     ```
     python -m venv .venv
     .\.venv\Scripts\Activate.ps1
     ```

3. 開発中はソースを直接参照する方法が簡単です：
   ```
   pip install -e .
   ```
   （プロジェクトに `setup.py` または `pyproject.toml` があることが前提です。ない場合は、プロジェクトルートを PYTHONPATH に追加するか、`src` 配下から直接実行してください。）

4. 依存パッケージがある場合は `requirements.txt` や `pyproject.toml` を参照してインストールしてください。

## 使い方（簡易例）

現状のパッケージはモジュールの骨組みのみのため、まずはインポートやバージョン確認から始めると良いです。

- パッケージインポート / バージョン確認
  ```python
  import kabusys

  print(kabusys.__version__)  # 0.1.0
  ```

- モジュール参照の例
  ```python
  from kabusys import data, strategy, execution, monitoring

  # 各モジュールに実装を追加して使用します。
  # 例: data.fetch_market_data(), strategy.generate_signals(), execution.send_order(), monitoring.report()
  ```

- 開発の流れ（例）
  1. `data` モジュールに市場データ取得関数を実装
  2. `strategy` モジュールにシグナル生成ロジックを実装
  3. `execution` モジュールで実際の発注API呼び出し（またはシミュレーション）を実装
  4. `monitoring` モジュールでログ・アラート・メトリクスを追加

## ディレクトリ構成

現在の最小構成（抜粋）:
```
.
├── src/
│   └── kabusys/
│       ├── __init__.py           # パッケージ初期化、__version__ 等
│       ├── data/
│       │   └── __init__.py       # データ取得ロジックを配置
│       ├── strategy/
│       │   └── __init__.py       # 売買戦略ロジックを配置
│       ├── execution/
│       │   └── __init__.py       # 発注（Execution）ロジックを配置
│       └── monitoring/
│           └── __init__.py       # 監視 / ログ / メトリクス
```

主要ファイル（抜粋）:
- src/kabusys/__init__.py
  - パッケージ名、docstring、`__version__`、`__all__` が定義されています。

## 実装ガイド（短く）

- data:
  - 市場データ取得関数やキャッシュ、レート制限処理をここに実装します。
- strategy:
  - シグナル生成、ポジション管理ルール、パラメータ化をここに実装します。
- execution:
  - 注文送信、注文状態の管理、約定確認の処理を実装します。証券会社APIラッパーはここに置くと良いです。
- monitoring:
  - ログ、メトリクス送信、アラート、Webダッシュボード連携等をここに実装します。

各モジュール内でインターフェースを決めておくと、ユニットテストやバックテストとの切替が容易になります。

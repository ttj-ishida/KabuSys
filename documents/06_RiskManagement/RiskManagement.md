# RiskManagement.md

## 1. 目的

本ドキュメントは、日本株自動売買システムにおける **リスク管理方針および制御ロジック** を定義する。

自動売買システムでは、戦略の収益性よりも **リスク管理の設計がシステムの安定性と長期運用を決定する**。

本ドキュメントの目的は以下である。

- 想定外の損失を防止する
- システム障害による誤発注を防ぐ
- 市場環境悪化時の資産保全を行う
- 自動売買を長期運用可能な状態に保つ

---

# 2. リスク管理の基本原則

本システムでは以下の原則を採用する。

1. **単一戦略に依存しない**
2. **ポジションサイズを制御する**
3. **損失は小さく、利益は伸ばす**
4. **市場環境に応じてリスク量を調整する**
5. **システム異常時は即座に取引停止する**

---

# 3. リスク分類

本システムでは以下のリスクを管理対象とする。

## 3.1 市場リスク

市場価格変動による損失

例

- 株価急落
- 市場クラッシュ
- セクター暴落

対策

- ポジションサイズ制御
- 損切りルール
- 市場レジーム判定

---

## 3.2 流動性リスク

流動性不足により売買が成立しないリスク

例

- 出来高不足
- スプレッド拡大
- 板薄銘柄

対策

- 最低売買代金フィルタ
- 板厚監視
- 成行注文制限

---

## 3.3 システムリスク

システム障害による誤動作

例

- APIエラー
- ネットワーク障害
- プログラム例外

対策

- フェイルセーフ設計
- 冪等注文ID
- キルスイッチ

---

## 3.4 モデルリスク

戦略モデルが市場環境に適合しなくなるリスク

例

- 市場レジーム変化
- 過剰最適化
- AI誤判定

対策

- バックテスト比較
- AIスコア影響制限
- レジーム制御

---

# 4. ポジションサイズ管理

## 4.1 1銘柄あたりの最大リスク

推奨

- 資産の **1〜2%**

例

資産

```
10,000,000 円
```

1トレード最大損失

```
100,000 円
```

---

## 4.2 ポートフォリオ制限

制限

| 項目               | 制限 |
| ------------------ | ---- |
| 最大保有銘柄       | 10   |
| 1銘柄最大比率      | 10%  |
| 同一セクター       | 30%  |
| キャッシュ最低比率 | 20%  |

---

# 5. 損切りルール

## 5.1 固定損切り

例

```
-5%
```

または

```
ATRベース
```

---

## 5.2 トレーリングストップ

利益が出ている場合

```
最高値 − ATR×2
```

---

# 6. 市場レジームによるリスク調整

市場を以下に分類する

| レジーム | リスク          |
| -------- | --------------- |
| Bull     | 通常            |
| Neutral  | やや縮小        |
| Bear     | 新規停止 / 縮小 |

Bear時

- 新規建停止
- 保有縮小
- キャッシュ増加

---

# 7. AI判断のリスク制御

AIスコアは以下を満たす必要がある

- 影響度 **最大10% (AI overlay limit = 10%)**
- confidence_score による減衰
- 異常値検出

AI異常時

```
news_score = 0
macro_score = 0
```

# 9. キルスイッチ

以下の条件で取引停止

- API連続エラー
- 異常ポジション
- 最大ドローダウン超過
- 注文数異常

発動時

```
新規発注停止
```

---

# 10. ドローダウン制御

| ドローダウン | 対応             |
| ------------ | ---------------- |
| 5%           | ポジション半減   |
| 10%          | 新規停止         |
| 15%          | 全ポジション縮小 |

# 15. まとめ

リスク管理は自動売買システムの最重要要素である。

本システムでは

- ポジションサイズ管理
- 損切りルール
- 市場レジーム制御
- AIリスク制御
- 発注安全機構
- キルスイッチ

を組み合わせることで、安全かつ長期的に運用可能な自動売買基盤を構築する。

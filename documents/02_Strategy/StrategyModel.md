# Strategy Model (売買戦略・シグナル生成基準)

- 対象: 日本株自動売買基盤におけるクオンツ戦略レイヤ
- 版数: v1.0

---

## 1. 目的

本仕様は、銘柄ユニバースの選定、特徴量（ファクター）の定義、およびそれらを統合した売買「シグナル」の生成・ポジションサイズ決定ルールを明文化するものである。
「AI Model (市場インテリジェンス層)」からのスコアを受け取り、最終的な期待値を算出して直近のトレードアクション（Entry/Exit）を決定する。

---

## 2. 銘柄ユニバースとスクリーニング

投資対象を絞り込み、kabuステーション APIのPUSH登録上限（最大50銘柄）という制約の中で安全な運用を行うための除外・監視ルール。

### 2.1 対象市場・母集団
- 東京証券取引所 プライム市場、スタンダード市場
- 除外対象: グロース市場、ETF、REIT、整理・監理銘柄

### 2.2 事前フィルタ（流動性・安全性）
最低限の流動性を担保し、スリッページや極端な値飛びを防ぐ。
- **最低株価**: 300円以上
- **流動性**: 20日平均売買代金 5億円以上
- **履歴期間**: 上場から250営業日以上経過していること

### 2.3 監視ユニバースの決定（日次）
前日夜間（シグナル生成時）に、各スコアの上位銘柄を抽出し、**翌営業日に監視する銘柄群（最大30〜40銘柄推奨）**をリスト化する。

---

## 3. 複合ファクター（スコアリング）仕様

ルールベースの定量指標を計算し、それぞれを正規化（0.0〜1.0 またはZスコア）して評価する。

### 3.1 モメンタム (Momentum)
- **指標**: 1ヶ月・3ヶ月・6ヶ月リターン、200日移動平均乖離率
- **傾向**: 順張り（トレンドフォロー）をベースとする。

### 3.2 バリュー (Value)
- **指標**: PER (株価収益率)、PBR (株価純資産倍率)、配当利回り
- **傾向**: テクニカルな過熱感を抑制し、下値不安の少ない割安銘柄を評価。

### 3.3 ボラティリティ & 流動性 (Volatility / Liquidity)
- **指標**: 20日ATR（Average True Range）、出来高変化率
- **傾向**: ボラティリティが適切な範囲にあり、直近で出来高急増などの変化があるか。

---

## 4. シグナル生成・スコア統合

各ファクターのスコアと、AIから支給される `news_score` を一定の重みで足し合わせ、最終的な期待値 `final_score` を算出する。

### 4.1 統合計算式（例）

```text
final_score = 
    0.40 * score_momentum
  + 0.20 * score_value
  + 0.15 * score_volatility
  + 0.15 * score_liquidity
  + 0.10 * adjusted_news_score (※最大影響度 10%に制限)
```

この `final_score` が所定の閾値 (`signal_threshold`) を超えた銘柄に対し、「買い（BUY）」シグナルを生成する。

---

## 5. エントリーとイグジットの基準

### 5.1 エントリー（Entry）
- **条件**: `final_score` が閾値以上であること。
- **レジームフィルタ**: AIが判定する `regime` が Bear（弱気・下落トレンド）の時は、どんなにスコアが高くても**新規の買いエントリーは生成しない（見送り）**。
- **執行タイミング**: 原則として「翌営業日の寄付き成行」または「寄付き指値」。

### 5.2 イグジット（Exit / 手仕舞い）
以下のいずれかに抵触した時点で、翌営業日に売却シグナルを発火する。
1. **ストップロス（絶対防衛ライン）**: 買値から -8% の下落、または 2×ATR 分の下落。
2. **トレーリングストップ（利益確保）**: 直近の最高値から -10% の下落。
3. **時間決済**: 保有から 60営業日（約3ヶ月）を経過。
4. **スコア低下**: 日次の計算で `final_score` が上位圏外へ転落。

---

## 6. ポジションサイジング（資金配分）とリスク額

1トレードにおける損失額の上限を口座総資産から逆算し、「何株買うべきか」を決定する。

### 6.1 リスク許容度
- 1回のトレードの許容リスク（損失額）: 総資産の **0.5% 〜 1.0%**
- 1銘柄の最大投資比率上限: 総資産の **10%** 
- 全ポジションの現金比率上限: 投下資金は最大で総資産の **70%** まで（残り30%はキャッシュ維持）

### 6.2 株数計算式

```text
1. 許容損失額 = 口座総資産 × risk_pct       （デフォルト: 0.005 = 0.5%）
2. 1株当たりリスク = エントリー株価 × stop_loss_pct （デフォルト: 0.08 = 8%ストップロス）
3. 算出株数 = floor( 許容損失額 / 1株当たりリスク )

# 上限クリップ: min(算出株数, floor(総資産 × max_position_pct / エントリー株価))
#              max_position_pct デフォルト: 0.10（10%）
# 単元株数（100株）単位に切り捨て
```

これにより、高ボラティリティな値嵩株は少なめに、低ボラティリティな銘柄は多めに買われる設計となり、ポートフォリオ全体のリスク（変動幅）が平滑化される。

実装パラメータ（`run_backtest` / 本番実行系共通）:

| パラメータ | デフォルト値 | 説明 |
|-----------|------------|------|
| risk_pct | 0.005 | 1トレード許容リスク率 |
| stop_loss_pct | 0.08 | 損切り率（株数計算用） |
| max_position_pct | 0.10 | 1銘柄上限（総資産比） |
| max_utilization | 0.70 | 全ポジション投下上限 |
| lot_size | 100 | 単元株数 |

実装モジュール: `src/kabusys/portfolio/position_sizing.py`（`allocation_method="risk_based"` 時）

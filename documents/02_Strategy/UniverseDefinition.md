
# UniverseDefinition.md

## 1. 目的

本ドキュメントは、日本株自動売買システムにおける **投資ユニバース（Universe）** の定義方法を規定する。

投資ユニバースとは、**戦略が分析対象とする銘柄集合**を意味する。

適切なユニバース設計は以下に直結する。

- 戦略の再現性
- 流動性確保
- スリッページ低減
- バックテストの現実性
- システム安定性

クオンツ運用では

```
Universe = Strategy Performance の基盤
```

と考えられる。

---

# 2. ユニバース設計の基本原則

本システムでは以下の原則を採用する。

1. **十分な流動性を持つ銘柄のみを対象とする**
2. **バックテストと本番で同一ユニバースを再現可能にする**
3. **銘柄入替ルールを明確にする**
4. **極端に小型の銘柄は除外する**
5. **データ取得が安定している銘柄のみ採用する**

# 3. 対象市場

基本対象

| 市場 | 対象 |
|-----|------|
東証プライム | ○ |
東証スタンダード | ○ |
東証グロース | △（流動性条件あり） |

```
東証プライムのみ
```

理由

- 流動性が高い
- 板が厚い
- スリッページが小さい

# 4. 銘柄マスタ

ユニバース管理は **銘柄マスタテーブル**で管理する。

テーブル

```
stocks
```

例

| column | 内容 |
|------|------|
code | 銘柄コード |
name | 銘柄名 |
market | 市場 |
sector | セクター |
listed_date | 上場日 |
delisted_date | 上場廃止日 |

---

# 5. 流動性フィルタ

以下の条件を満たす銘柄のみ採用する。

## 最低売買代金

```
5億円以上 / 日
```

## 最低出来高

```
10万株 / 日
```

## 株価

```
300円以上
```

理由

- 低位株排除
- 板薄銘柄排除

---

# 6. データ品質フィルタ

以下の条件を満たす銘柄のみ採用する。

- 過去データ欠損が少ない
- 株価履歴が一定期間存在
- 財務データ取得可能

例

```
過去250営業日データ存在
```

# 11. 除外ルール

以下の銘柄は除外する。

- 整理銘柄
- 監理銘柄
- 極端な低流動性銘柄
- 長期取引停止銘柄

---

|------|------|

---

# 13. バックテスト整合性

バックテストでは以下を守る。

- 過去時点のユニバースを再現
- 上場廃止銘柄を除外しない
- 生存バイアスを防ぐ

重要

```
Survivorship Bias を防ぐ
```

# 16. まとめ

ユニバース設計は以下のプロセスで構築する。

```
銘柄マスタ
↓
市場フィルタ
↓
流動性フィルタ
↓
品質チェック
↓
ユニバース生成
```

本設計により、**流動性と再現性を確保した投資対象銘柄集合**を構築する。

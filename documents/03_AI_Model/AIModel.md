# AI Model & Market Intelligence (人工知能・市場インテリジェンス層)

- 対象: ニュース自然言語処理(NLP)、マクロ指標スコアリング
- 版数: v1.0

---

## 1. AIモデルの役割と基本原則

本システムにおけるAI（LLM等）の役割は、**「テキストデータや非定型なマクロ情報の定量化（スコア化）」**に限定する。
AIが直接「買う・売る」の判断を下したり、APIを叩いて直接発注指示を出すことは**絶対にない**。

すべては数値（スコアや判定フラグ）として出力され、`01_Data` 層に保存されたのち、`02_Strategy` 層へ渡されてルールベースで統合評価される。これにより「なぜその取引が行われたか」という監査性（Auditability）と再現性を維持する。

## 2. ニュースセンチメント分析 (NLP)

企業の適時開示やニューストピックから、その日のセンチメント（市場心理）を判定する。

### 2.1 入力データ
- J-Quants ニュースAPI、適時開示情報、または外部ニュースのヘッドライン
- 前日の大引け後（15:00）〜 当日の寄り付き前（8:30）に発生したニューステキスト

### 2.2 処理フロー
1. **エンティティ抽出**: ニュース本文から言及されている企業名（銘柄コード）を特定する。
2. **要約・極性判定**: テキストをLLM（例: Gemini, GPT-4等）に渡し、「ポジティブ(1.0)・ニュートラル(0.0)・ネガティブ(-1.0)」の極性、あるいは `-1.0 〜 1.0` の連続値スコアで評価させる。
3. **JSON出力**: `{"symbol": "7203", "sentiment_score": 0.85, "summary": "業績上方修正..."}` の形式で出力し、Data Layerへ保存する。

### 2.3 フィルタリングと制限
- センセーショナルな見出しによる誤判定を防ぐため、1記事への過信を避け、複数情報の平均スコアや移動平均を優先する。
- Strategy層での重み付けは最大でも全体の 10%〜20% 程度に制限し、財務・テクニカル指標の裏付けがないニュース単独での高額エントリーを防ぐ。

## 3. マクロイベントと市場レジーム判定

個別銘柄のセンチメントだけでなく、市場全体が現在「買い向かうべきか、撤退すべきか」の環境（レジーム）を判定する。

### 3.1 レジームの種類
1. **Bull (ブル・強気)**: 上昇トレンド。積極的な買い戦略を許可する。
2. **Neutral (ニュートラル・中立)**: ボックス相場。標準的なポジションサイズで運用。
3. **Bear (ベア・弱気)**: 下落トレンド、ボラティリティ急騰時など。**新規買いシグナルをすべて遮断・無効化する。** キャッシュ比率を高める。

### 3.2 判定要素（実装済み）
- **ETF 1321（日経225連動型）の200日移動平均との乖離（重み70%）**: `prices_daily` から取得。
  `ma200_ratio = 終値 / 200日MA`
- **マクロ経済ニュースの LLM センチメント（重み30%）**: `raw_news` からキーワード
  （日銀・FOMC・CPI・為替介入等）でフィルタし、gpt-4o-mini で評価。

将来対応（別 Issue）:
- VIX指数 / 日経VIの閾値超え

### 3.3 スコア合成
```
regime_score = clip(0.7 * (ma200_ratio - 1.0) * 10 + 0.3 * macro_sentiment, -1.0, 1.0)
```
- `score >= +0.2` → 'bull'
- `score <= -0.2` → 'bear'
- それ以外 → 'neutral'

### 3.4 実装モジュール
- `src/kabusys/ai/regime_detector.py` — `score_regime(conn, target_date, api_key)`
- 出力先: `market_regime` テーブル（日付単位・1行）
- `ai_scores.regime_score` は使用しない（NULL のまま保持）

## 4. 異常時・フェイルセーフ仕様

AIはクラウドAPIの制約やハルシネーションにより、ブラックボックス化しやすい性質を持つ。API障害や出力フォーマット崩れ（JSONパースエラー等）が発生した場合の振る舞いを厳格に定義する。

- **フォールバック（縮退運転）**: AIから有効なスコアが取得できない場合、システムは異常終了せず、その銘柄（または全銘柄）の AIスコアを強制的に `0.0 (ニュートラル)` として扱う。
- 財務・テクニカルスコアのみでの運用（ルールベース戦略）に一時的に自動縮退し、取引機会の完全な逸失を防ぎつつ、意図せぬ暴走を隔離する。

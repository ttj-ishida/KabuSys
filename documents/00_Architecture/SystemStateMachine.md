# SystemStateMachine.md

## 1. 目的

本ドキュメントは、日本株自動売買システムの **システム状態遷移（System
State Machine）** を定義する。

目的:

-   システムの実行状態を明確化
-   Runtime動作の理解を容易にする
-   障害時の挙動を定義する
-   自動売買の安全性を高める

対象環境:

-   Single Windows Node
-   Python Trading System
-   kabuステーションAPI
-   J-Quants Data

------------------------------------------------------------------------

# 2. システム状態一覧

システムは以下の状態を持つ。

  State               説明
  ------------------- --------------------
  INIT                システム初期化
  NIGHT_BATCH         夜間データ処理
  SIGNAL_GENERATION   シグナル生成
  PORTFOLIO_BUILD     ポートフォリオ構築
  PRE_MARKET          市場開始前準備
  TRADING             ザラ場取引
  MARKET_CLOSE        市場終了処理
  IDLE                待機
  RECOVERY            障害復旧
  KILL_SWITCH         強制停止

------------------------------------------------------------------------

# 4. 状態詳細

## INIT

システム初期化。

処理

-   config読み込み
-   DB接続
-   API接続確認

次状態

    NIGHT_BATCH

------------------------------------------------------------------------

## NIGHT_BATCH

夜間バッチ処理。

処理

-   市場データ取得
-   ニュース取得
-   特徴量生成

参照

    DataPlatform

次状態

    SIGNAL_GENERATION

------------------------------------------------------------------------

## SIGNAL_GENERATION

売買シグナル生成。

処理

-   Strategy計算
-   AIスコア統合

出力

    signals

次状態

    PORTFOLIO_BUILD

------------------------------------------------------------------------

## PORTFOLIO_BUILD

ポートフォリオ構築。

処理

-   銘柄ランキング
-   ポジションサイズ計算

出力

    signal_queue

次状態

    PRE_MARKET

------------------------------------------------------------------------

## PRE_MARKET

市場開始前処理。

処理

-   Execution Engine起動
-   Signal Queue読み込み
-   API接続確認

次状態

    TRADING

------------------------------------------------------------------------

## TRADING

市場取引状態。

処理

-   シグナル取得
-   発注
-   約定管理
-   ポジション更新

参照

    ExecutionSystem

次状態

    MARKET_CLOSE

------------------------------------------------------------------------

## MARKET_CLOSE

市場終了処理。

処理

-   ポジション更新
-   パフォーマンス計算
-   ログ保存

更新

    portfolio_performance

次状態

    IDLE

------------------------------------------------------------------------

## IDLE

待機状態。

処理

-   次NightBatchまで待機

次状態

    NIGHT_BATCH

------------------------------------------------------------------------

## RECOVERY

障害復旧。

処理

-   ログ確認
-   DB整合性チェック
-   ポジション確認

次状態

    INIT

------------------------------------------------------------------------

## KILL_SWITCH

緊急停止。

トリガー

-   最大ドローダウン
-   API接続断
-   Execution異常

処理

    注文停止
    Execution停止
    通知送信

復旧

    Manual Restart

------------------------------------------------------------------------

# 5. 状態遷移ルール

状態遷移は以下に従う。

1.  時刻ベース遷移（Market Schedule）
2.  イベントベース遷移（Signal生成）
3.  エラーベース遷移（Recovery）

------------------------------------------------------------------------

# 6. Runtimeとの関係

SystemStateMachine は Runtime設計と連動する。

    RuntimeArchitecture
    RuntimeJobSchedule

Runtime Scheduler は現在の状態に応じてジョブを起動する。

------------------------------------------------------------------------

# 7. まとめ

システム状態遷移

    INIT
     ↓
    NIGHT_BATCH
     ↓
    SIGNAL_GENERATION
     ↓
    PORTFOLIO_BUILD
     ↓
    PRE_MARKET
     ↓
    TRADING
     ↓
    MARKET_CLOSE
     ↓
    IDLE

障害時

    RECOVERY
    または
    KILL_SWITCH

この状態機械により **安全かつ制御可能な自動売買システム**を実現する。
